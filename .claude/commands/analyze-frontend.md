# Command — Analyze Frontend

Use this command when starting frontend work.

## Goal

Analyze `rajmango-web` without changing application code.

## Steps

1. Read:
   - `CLAUDE.md`
   - `README.md`
   - `docs/conventions/frontend-angular.md`
   - `docs/conventions/clean-code.md`
   - `.claude/rules/frontend.md`

2. Inspect:
   - Angular module structure
   - routing
   - Metronic layout integration
   - auth guards/interceptors
   - API services/clients
   - forms and validation
   - dashboard/screens
   - models and DTO duplication
   - build scripts

3. Produce:
   - current frontend architecture summary
   - launch-critical gaps
   - UI/UX risks
   - API integration risks
   - security/route-guard risks
   - prioritized execution plan

## Output Format

```text
Frontend Summary:
Launch Gaps:
UI/UX Risks:
API Integration Risks:
Security Risks:
Critical Tasks:
Important Tasks:
Nice-to-have:
Recommended Next Step:
```

Do not modify source code during this command.
